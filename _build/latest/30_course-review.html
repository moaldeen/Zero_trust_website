Machine learning workflow#

Learning objectives#

Here are the course learning outcomes:

  1. Identify problems that may be addressed with machine learning.

  2. Select the appropriate machine learning tool for a task/problem.

  3. Transform data of various types into usable features.

  4. Apply standard tools implementing supervised and unsupervised learning techniques.

  5. Describe core differences between training, validation, and testing regimes.

  6. Be realistic about the limitations of specific approaches and machine learning.

  7. Identify and avoid scenarios where training and testing data are accidentally mixed (the “Golden Rule”).

  8. Employ good habits for applying ML, such as starting an analysis with a baseline estimator.

  9. Create responsible machine-learning solutions and products.

What did we cover?#

This course is roughly organized like this (not in order):

Part 1: Supervised learning on tabular data

  • Overfitting, train/validation/test/deployment, cross-validation

  • Feature preprocessing, pipelines, imputation, OHE, etc

  • The Golden Rule, various ways to accidentally violate it

  • Classification metrics: confusion matrix, precision/recall, ROC, AUC

  • Regression metrics: MSE, MAPE

  • Hyperparameter optimization

  • A bunch of models:

    • Baselines

    • Linear models (ridge, logistic regression)

    • KNNs and RBF SVMs

    • Tree-based models (random forest, gradient-boosted trees)

    • Ensembles

Part 2: Other tasks

  • Unsupervised learning

    • Dimension reduction: PCA

    • Clustering: K-Means, DBSCAN

  • Recommender systems

  • Trustworthy machine learning

Some key takeaways#

Some useful guidelines:

  • Do a train-test split right away and only once

  • Don’t look at the test set until the end

  • Don’t call fit on test/validation data

  • Use transformers and pipelines

  • Use baselines

Recipe to approach a supervised learning problem with tabular data#

  1. Have a long conversation with the stakeholder(s) who will be using your pipeline.

  2. Have a long conversation with the person(s) who collected the data.

  3. Think about the ethical implications - are you sure you want to do this project? If so, should ethics guide your approach?

  4. Random train-test split with fixed random seed; do not touch the test data until final evaluation.

  5. Exploratory data analysis, outlier detection.

  6. Choose a scoring metric –> higher values should make you and your stakeholders happier.

  7. Fit a baseline model, e.g., DummyClassifier or DummyRegressor.

  8. Create a pre-porcessing pipeline. This may involve feature engineering. (This is usually a time-consuming step.)

  9. Try a linear model. For example, LogisticRegression or Ridge; tune hyperparameters with CV.

  1. Try other sensible models(s), e.g., LightGBM; tune hyperparameters with CV.

  2. For each model, look at sub-scores from the folds of cross-validation to get a sense of “error bars” on the scores.

  3. Pick a promising model. The best CV score is a reasonable metric, though you may choose to favor simpler models.

  4. Look at feature importances.

  5. (optional) Perform some more diagnostics like a confusion matrix for classification or “predicted vs. true” scatterplots for regression.

  6. (optional) Try to calibrate the uncertainty or confidence outputted by your model.

  7. Test set evaluation.

  8. Question everything again: validity of results, bias/fairness of trained model, etc.

  9. Discuss your results with stakeholders.

  10. (optional) Retrain models on all your data.

  11. Deployment and integration.

  12. Profit?

conda, minoconda, and anaconda#

Conda is an excellent tool for organizing your Python programming environment. It efficiently manages packages and creates virtual environments for your projects, ensuring they do not interfere with one another. By using Conda, you can establish separate environments, each equipped with the necessary packages for a specific project, simplifying management and enhancing organization.

Building upon the capabilities of Conda, there are two notable Python distributions: Miniconda and Anaconda.

The simple way - Anaconda#

Please follow the instructions to install Anaconda on your computer.

What is Anaconda Distribution?

conda + Anaconda Navigator + additional packages for machine learning and data science.

  • conda: a package manager

  • Anaconda Navigator: GUI for managing conda packages, environments, and channels

  • packages: ready to use for common tasks

You can install 8000 extra packages.

BTW: Clemson’s Palmetto Cluster has Anaconda installed by default. You can use it whenever you are affiliated with Clemson.

The geek way - miniconda#

Please follow the instructions to install miniconda on your computer.

miniconda is a free minimal installer for conda. It is a small bootstrap version of Anaconda that includes only conda, Python, the packages they both depend on, and a small number of other useful packages (like pip, zlib, and a few others).

Create a virtual environment from Anaconda or Miniconda#

  • A virtual environment is a named, isolated, working copy of Python that maintains its own files, directories, and paths so that you can work with specific versions of libraries or Python itself without affecting other Python projects.

  • Virtual environments make it easy to separate different projects and avoid problems with various dependencies and version requirements across components.

  • The conda command is the preferred interface for managing installations and virtual environments with the Anaconda or miniconda Python distribution.

You can

  • Create an environment from scratch with commands. Instructions

  • Create an environment from a yml file. Instructions

    • A yml file includes channels, packages, and versions. (no platform information)

    • An example of a yml file.

  • Build an identical environment from a txt file. Instructions

    • You can use explicit specification files to build an identical conda environment on the same operating system platform, either on the same machine or on a different machine.

AI career pathway#

Developing an AI project development life cycle involves five distinct tasks:

  • Data engineering: People responsible for data engineering prepare data and transform data into formats that other team members can use.

  • Modeling: People assigned to modeling look for patterns in data that can help a company predict outcomes of various decisions, identify business risks and opportunities, or determine cause-and-effect relationships.

  • Deployment: People in charge of deployment take a stream of data, combine it with a model, and test the integration before putting the model into production.

  • Business analysis: Team members responsible for business analysis evaluate a deployed model’s performance and business value and adjust accordingly to maximize benefit or abandon unproductive models.

  • AI infrastructure: People who work in AI infrastructure build and maintain reliable, fast, secure, and scalable software systems to help people working in data engineering, modeling, deployment and business analysis.
