Setup#

  • Update: 2023

  • Duration: 75 minutes

  1. This page illustrates how to create a Python environment on the Palmetto cluster and launch a JupyterLab on OpenOnDemand.

  2. Please do not start your configuration before the in-class lecture.

  3. Palmetto servers experience higher demand at the end of each semester so please plan your homework and project in advance!

Why Use Palmetto Servers?#

There are three fundamental reasons we will be centralizing our work on the Palmetto servers:

Complex Dependencies: The scope of our course encompasses the use of over a hundred packages. This translates to intricate dependencies and potential conflicts. To ensure a seamless experience, the teaching team have worked diligently to devise a solution tailored to the Palmetto environment. While you might be inclined to replicate this setup on your personal computers, you are strongly advised against it. The intricacy can lead to numerous issues, and troubleshooting them might detract from the primary learning objectives.

Consistent Results & Grading: The teaching team have tested the lectures and homework rigorously in the Palmetto environment. By working on this platform, you are guaranteed to achieve consistent results as your peers. More importantly, it ensures that your homework gets graded in an identical environment, substantially minimizing any technical hitches.

Collaboration Made Easier: Many of you will be collaborating with peers on your final projects. Centralizing our work on Palmetto ensures everyone is on the same page, technically speaking. It simplifies the collaboration process, making sharing, and integration a breeze.

Warning

You are strongly advised against replicating this setup on your personal computers.

Create a Python environment at Palmetto#

Warning

This is a one-time setup, unless in the rare circumstance you need to reinstall Python at a later date.

  1. Go to Palmetto OpenOnDemand and sign in.

  2. Click on the Clusters tab on the left top and select Palmetto Shell Access.

  1. In the SSH page, type the following command to request hardware resources interactively.

    qsub -I -l select=1:ncpus=2:mem=8gb:interconnect=10ge,walltime=1:00:00

  2. Type the following command to load Anaconda.

    module load anaconda3/2022.10-gcc/9.5.0

  1. Type the following command to download the configuration file.

    wget -O ece4420.txt https://yongkaw.people.clemson.edu/ece4420/ece4420.txt

  2. Type the following command to create a conda environment called ece4420 at Palmetto.

    CONDA_PKGS_DIRS=$(mktemp -d) conda create --name ece4420 --file ece4420.txt

  3. Type the following command to activate the conda environment:

    source activate ece4420

  4. Type the following command to test the cmle environment installation in Python.

    pip show cmle

  5. The screen shows a summary about cmle rather than Warning: package(s) not found.

Warning

Please ensure you complete the above steps correctly before move forward.

Start Jupyter Notebook from Open OnDemand#

If you want to write code using JupyterLab, the browser-based notebook is a better choice.

Warning

Every time you use Jupyter Notebook or JupyterLab, you need to request hardware resources using the following template. Fortunately, the template will load your last setting and save your time. But sometimes the template will be reset. You can slightly adjust the template based on your demand.

Start an Jupyter instance via Palmetto OpenOnDemand#

  • Go to Palmetto OpenOnDemand and sign in.

  • Under Interactive Apps tab, select Jupyter Notebook.

  • Make the following selections:

    • Anaconda Version: anaconda3/2022.05-gcc/9.5.0

    • Path to Python virtual/conda environment: source activate ece4420

  • Make the remaining selections according to how many resources you would need.

    • Number of resource chunks: 1

    • CPU cores per chunk: 8

    • Amount of memory per chunk: 10gb

    • Number of GPUs per chunk: None

    • GPU Model: None

    • Interconnect: any

    • Walltime: 02:00:00

  • Click Launch when done.

Connect to the JypyterNotebook#

Once the JupyterNotebook app has started successfully, you are redirected to a new page. Please wait for the Connect to Jupyter button. If it takes a long while and the button is not available, it might incur errors.

As the Jupyter Server is launched directly out of the JupyterLab package installed in the conda environment, no special kernel is needed.

FAQs#

Modules are unknown#

Description#

When you attempt to load the anaconda module but receive a message saying anaconda is unknown.

$ module load anaconda3/2022.10-gcc/9.5.0
Lmod has detected the following error: The following module(s) are unknown: "anaconda3/2022.10-gcc/9.5.0"

Solution#

You cannot load the module as your account is locked. Please visit or call the CCIT Help Desk and they can assist you in unlocking your account. See https://ccit.clemson.edu/support/ for the helpdesk hours, location, and phone number.

An environment already exists#

Description#

$ CONDA_PKGS_DIRS=$(mktemp -d) conda create --name ece4420s --file ece4420.txt
Retrieving notices: ...working... done
WARNING: A conda environment already exists at '/home/yongkaw/.conda/envs/ece4420'
Remove existing environment (y/[n])?

Solution#

An environment already exists at the designated location. This could indicate a previous setup that might be incorrect. Stop the installation by press n. Then remove the existing environment using the following command.

conda remove --name ece4420 --all -y

A directory already exists#

Description#

$ CONDA_PKGS_DIRS=$(mktemp -d) conda create --name ece4420s --file ece4420.txt
WARNING: A directory already exists at the target location '/home/yongkaw/.conda/envs/ece4420'
but it is not a conda environment.
Continue creating environment (y/[n])?

Solution#

A directory already exists at the intended environment path. This can suggest an interrupted or incomplete setup. Stop the installation by press n. Then remove the existing directory using the following command.

rm -rf .conda/envs/ece4420

No module named ‘cmle`#

Description#

After successfully setting up the development environment, students have been utilizing it for their projects without any issues. However, unexpectedly, they encountered a system error indicating that no module named ‘cmle’.

Solution#

The issue of the ‘cmle’ package becoming unavailable in the Palmetto Cluster may be due to system updates and routine clearing of history by system administrators. When history is cleared, all settings configured through Open OnDemand are lost. To resolve this, students are advised to revisit the setup instructions for Open OnDemand. It is crucial to carefully follow these instructions and ensure all fields are correctly filled during the setup process.

Install miniconda#

Please follow the instructions to install miniconda in your computer.

miniconda is a free minimal installer for conda. It is a small bootstrap version of Anaconda that includes only conda, Python, the packages they both depend on, and a small number of other useful packages (like pip, zlib, and a few others).

BTW: conda, minoconda, and anaconda

Conda is an excellent tool for organizing your Python programming environment. It efficiently manages packages and creates virtual environments for your projects, ensuring they do not interfere with one another. By using Conda, you can establish separate environments, each equipped with the necessary packages for a specific project, simplifying management and enhancing organization.

Building upon the capabilities of Conda, there are two notable Python distributions: Miniconda and Anaconda.

Create a conda environment#

Please download an environment yaml file. (Should wget be unavailable, kindly download the file manually and save it with the filename ece4420.yml.)

wget -O ece4420.yml https://yongkaw.people.clemson.edu/ece4420/ece4420.yml

Create the environment from the environment file

conda env create --name ece4420 --file ece4420.yml

Activate the environment and install additional packages#

Activate the conda environment

activate ece4420

Type the following commands to install additional packages.

conda install https://yongkaw.people.clemson.edu/ece4420/packages/mglearn-0.2.1.1-py_0.tar.bz2
